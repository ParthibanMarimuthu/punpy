50 """ 

51 Propagate random uncertainties through measurement function with n input quantities. 

52 Input quantities can be floats, vectors (1d-array) or images (2d-array). 

53 Random uncertainties arise when there is no correlation between repeated measurements. 

54 It is possible (though rare) that there is a correlation in one of the dimensions that is not one of the repeat_dims. 

55 

56 :param func: measurement function 

57 :type func: function 

58 :param x: list of input quantities (usually numpy arrays) 

59 :type x: list[array] 

60 :param u_x: list of random uncertainties on input quantities (usually numpy arrays) 

61 :type u_x: list[array] 

62 :param corr_x: list of correlation matrices (n,n) along non-repeating axis, defaults to None. Can optionally be set to "rand" (diagonal correlation matrix), "syst" (correlation matrix of ones) or a custom correlation matrix. 

63 :type corr_x: list[array], optional 

64 :param param_fixed: when repeat_dims>=0, set to true or false to indicate for each input quantity whether it has repeated measurements that should be split (param_fixed=False) or whether the input is fixed (param fixed=True), defaults to None (no inputs fixed). 

65 :type param_fixed: list of bools, optional 

66 :param corr_between: correlation matrix (n,n) between input quantities, defaults to None 

67 :type corr_between: array, optional 

68 :param return_corr: set to True to return correlation matrix of measurand, defaults to False 

69 :type return_corr: bool, optional 

70 :param return_samples: set to True to return generated samples, defaults to False 

71 :type return_samples: bool, optional 

72 :param repeat_dims: set to positive integer(s) to select the axis which has repeated measurements. The calculations will be performed seperately for each of the repeated measurments and then combined, in order to save memory and speed up the process. Defaults to -99, for which there is no reduction in dimensionality.. 

73 :type repeat_dims: integer or list of 2 integers, optional 

74 :param corr_axis: set to positive integer to select the axis used in the correlation matrix. The correlation matrix will then be averaged over other dimensions. Defaults to -99, for which the input array will be flattened and the full correlation matrix calculated. 

75 :type corr_axis: integer, optional 

76 :param fixed_corr_var: set to integer to copy the correlation matrix of the dimension the integer refers to. Set to True to automatically detect if only one uncertainty is present and the correlation matrix of that dimension should be copied. Defaults to False. 

77 :type fixed_corr_var: bool or integer, optional 

78 :param output_vars: number of output parameters in the measurement function. Defaults to 1. 

79 :type output_vars: integer, optional 

80 :param PD_corr: set to True to make sure returned correlation matrices are positive semi-definite, default to True 

81 :type PD_corr: bool, optional 

82 :return: uncertainties on measurand 

83 :rtype: array 

84 """
